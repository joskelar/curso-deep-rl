5.2 · Paso 1: Setup

Paso 1: Preparar ML-Agents en Colab 🔧

El notebook requiere Python 3.10 — Colab a veces tiene una versión incompatible. El script crea un entorno virtual con la versión correcta:

# 1. Clonar ML-Agents desde GitHub (puede tardar ~3 min)
!git clone --depth 1 https://github.com/Unity-Technologies/ml-agents

# 2. Instalar Miniconda y crear entorno Python 3.10
!wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
!./Miniconda3-latest-Linux-x86_64.sh -b -f -p /usr/local
!conda install -q -y --prefix /usr/local python=3.10.12

# 3. Instalar ML-Agents desde el código fuente
%cd ml-agents
!pip3 install -e ./ml-agents-envs
!pip3 install -e ./ml-agents

⚠️ Error frecuente: "command not found"

Si ves mlagents-learn: command not found significa que la instalación no coincidió con la versión de Python de Colab. El script de Miniconda resuelve esto creando un entorno con Python 3.10.12 explícitamente.

💡 La bandera --depth 1

ML-Agents tiene un historial de commits enorme. --depth 1 clona solo el estado más reciente (sin historial), lo que reduce el tiempo de descarga de ~15 min a ~3 min.

5.2 · Paso 2: Ejecutable

Paso 2: Descargar el entorno de Unity 📦

El entorno de SnowballTarget viene como un ejecutable de Unity compilado para Linux (ya que Colab corre Ubuntu). No necesitamos Unity instalado — solo el binario:

# Crear carpetas para los ejecutables
!mkdir -p ./training-envs-executables/linux/

# Descargar SnowballTarget.zip desde GitHub
!wget "https://github.com/huggingface/Snowball-Target/raw/main/SnowballTarget.zip" \
  -O ./training-envs-executables/linux/SnowballTarget.zip

# Descomprimir el archivo
!unzip -d ./training-envs-executables/linux/ \
  ./training-envs-executables/linux/SnowballTarget.zip

# Dar permisos de ejecución (necesario en Linux)
!chmod -R 755 ./training-envs-executables/linux/SnowballTarget

🐧 ¿Por qué un ejecutable Linux? Colab corre Ubuntu. El ejecutable de Unity ya contiene el motor de juego completo compilado para esa plataforma — la física, los gráficos, la lógica del juego. ML-Agents Python solo envía acciones y recibe observaciones a través de una conexión local.

5.2 · Paso 3: YAML

Paso 3: Configurar los hiperparámetros en YAML 📄

Crea el archivo ./config/ppo/SnowballTarget.yaml con estos parámetros. Aquí explicamos los más importantes:

behaviors:
  SnowballTarget:
    trainer_type: ppo
    summary_freq: 10000
    keep_checkpoints: 10
    max_steps: 200000
    time_horizon: 64
    threaded: true
    hyperparameters:
      learning_rate: 0.0003
      batch_size: 128
      buffer_size: 2048
      epsilon: 0.2
    network_settings:
      hidden_units: 256
      num_layers: 2

max_steps: 200,000
Pasos totales de entrenamiento. Para la meta (15 objetivos/episodio) son suficientes.

keep_checkpoints: 10
Guarda los últimos 10 modelos. Útil para comparar distintos momentos del entrenamiento.

time_horizon: 64
Cada cuántos pasos se actualiza la política aunque el episodio no haya terminado.

5.2 · Paso 4: Entrenamiento

Paso 4: ¡A entrenar! 🔥

Con el YAML listo, lanzamos el entrenamiento con un solo comando:

# Lanzar entrenamiento (~10-35 min según GPU)
!mlagents-learn ./config/ppo/SnowballTarget.yaml \
  --env=./training-envs-executables/linux/SnowballTarget/SnowballTarget \
  --run-id="SnowballTarget1" \
  --no-graphics

# Si Colab se desconecta, usa --resume para retomar
# !\mlagents-learn ... --resume (puede fallar la 1a vez, ejecutar 2 veces)

ML-Agents corre múltiples copias del entorno en paralelo dentro del mismo ejecutable. Todos los agentes comparten la misma política pero juegan episodios independientes. Esto acelera el entrenamiento.

☕ Descansa un momento. El entrenamiento tarda entre 10 y 35 minutos según la GPU. Puedes ver el progreso en los logs: la línea Mean Reward debería ir subiendo gradualmente. Meta: recompensa media ≥ 15 objetivos por episodio.
