2.5 · Versión Resbaladiza

¡El lago resbala! ¿Versión resbaladiza o no? 🛝

FrozenLake tiene dos modos. En el curso de HuggingFace usamos la versión no-resbaladiza primero para facilitar el aprendizaje:

✅ No resbaladiza

is_slippery=False

El agente se mueve exactamente donde quiere. Si elige ir a la derecha, va a la derecha. Sin sorpresas.

🧝 → elige ➡️ → 🧝 siempre ➡️ ✅

⚠️ Resbaladiza (el reto)

is_slippery=True

El agente elige una dirección, pero el hielo lo puede desviar a los lados. Solo tiene 33% de probabilidad de ir adonde quiere.

🧝 → elige ➡️ → puede ir ➡️, ⬆️ o ⬇️
🎯 Estrategia del curso

Empezamos con is_slippery=False para que el agente aprenda la política perfecta sin ambigüedad. Luego el Capítulo 2.6 usa Taxi-v3, que tiene un espacio de estados mucho mayor como reto adicional.

2.5 · Código: Q-table y políticas

Inicializar la Q-table y definir las políticas 🗂️

# ── Inicializar la Q-table con ceros ──────────────────
def inicializar_qtable(env):
  n_estados = env.observation_space.n
  n_acciones = env.action_space.n
  return np.zeros((n_estados, n_acciones)) # → (16, 4)

# ── Política óptima: siempre elige la mejor acción ───
def politica_optima(Qtable, estado):
  return np.argmax(Qtable[estado])

# ── Política ε-greedy: explora O explota ─────────────
def politica_epsilon_greedy(Qtable, estado, epsilon):
  if random.uniform(0, 1) > epsilon:
    return np.argmax(Qtable[estado]) # explotación
  else:
    return env.action_space.sample() # exploración
🔑 Las dos políticas

politica_optima se usa al evaluar el agente (sin explorar). politica_epsilon_greedy se usa durante el entrenamiento para equilibrar exploración y explotación.

2.5 · Código: Entrenamiento

El bucle de entrenamiento: donde todo ocurre 🔁

Esta función es el corazón del notebook. Contiene el algoritmo Q-Learning en ~20 líneas:

def entrenar(n_episodios, min_eps, max_eps, decay, env, max_pasos, Qtable, alpha, gamma):
  for episodio in tqdm(range(n_episodios)):
    # 1. Reducir ε conforme avanza el entrenamiento
    epsilon = min_eps + (max_eps - min_eps) * np.exp(-decay * episodio)
    estado, _ = env.reset()
    terminado = False

    for paso in range(max_pasos):
      # 2. Elegir acción con ε-greedy
      accion = politica_epsilon_greedy(Qtable, estado, epsilon)

      # 3. Ejecutar acción
      nuevo_estado, recompensa, terminado, _, _ = env.step(accion)

      # 4. ★ Actualizar Q-table (la línea más importante)
      Qtable[estado, accion] += alpha * (
        recompensa + gamma * np.max(Qtable[nuevo_estado]) - Qtable[estado, accion]
      )
      if terminado: break
      estado = nuevo_estado
  return Qtable
2.5 · Hiperparámetros

Los hiperparámetros del entrenamiento ⚙️

Estos son los valores exactos que usa el curso de HuggingFace para FrozenLake. Después podrás experimentar cambiándolos:

# Parámetros de entrenamiento
n_episodios = 10_000 # 10,000 partidas
tasa_aprendizaje = 0.7 # α = 0.7 (bastante agresivo)
gamma = 0.95 # descuento γ
max_pasos = 99 # máximo de pasos por episodio

# Parámetros de exploración
max_epsilon = 1.0 # ε al inicio: 100% exploración
min_epsilon = 0.05 # ε mínimo: siempre 5% de exploración
decay_rate = 0.0005 # velocidad de decaimiento de ε

α = 0.7 — ¿Por qué tan alto?

FrozenLake no-resbaladizo es determinista. Con recompensas que no tienen ruido, podemos aprender más rápido con una tasa mayor.

10,000 episodios

Con 16 estados y sin resbalones, el agente converge rápido. Este número es más que suficiente.

decay = 0.0005

Decaimiento lento: el agente explora mucho durante los primeros miles de episodios antes de explotación.

2.5 · Evaluar y publicar

Evaluar el agente y publicarlo en el Hub 📊🤗

Una vez entrenado, evaluamos en 100 episodios y guardamos la Q-table en el Hub:

# Evaluación: correr 100 episodios sin exploración
def evaluar(env, Qtable, n_episodios=100):
  recompensas = []
  for _ in range(n_episodios):
    estado, _ = env.reset()
    total = 0
    for _ in range(200):
      accion = politica_optima(Qtable, estado)
      estado, r, done, _, _ = env.step(accion)
      total += r
      if done: break
    recompensas.append(total)
  return np.mean(recompensas), np.std(recompensas)

2.6 · Recompensas Taxi

Las recompensas de Taxi: muchas formas de equivocarse 💰

Taxi tiene un sistema de recompensas más rico que FrozenLake, diseñado para que el agente aprenda comportamientos específicos:

Recompensas positivas ✅

📤
+20 puntos
Dejar al pasajero en el destino correcto

Penalizaciones ❌

🚗
−1 punto
Cada paso de movimiento (incentiva la eficiencia)
⚠️
−10 puntos
Recoger o dejar pasajero en el lugar equivocado
🧠 Por qué −1 por cada paso

Si el agente solo recibiera +20 al final, podría aprender a dar vueltas infinitas antes de terminar — ¡seguiría ganando en algún momento! La penalización de −1 por paso lo obliga a llegar lo más rápido posible.

2.6 · Código

Taxi con el mismo código que FrozenLake 🔄

La gran ventaja de Q-Learning: el algoritmo es idéntico. Solo cambian los parámetros y el entorno. Esto demuestra que aprendiste algo general, no solo un truco específico:

# ── Cambios mínimos para Taxi ─────────────────────────

# 1. Cambiar el entorno
env = gym.make("Taxi-v3")
print("Estados:", env.observation_space.n) # → 500 (vs 16 de FrozenLake)
print("Acciones:", env.action_space.n) # → 6 (vs 4 de FrozenLake)

# 2. La Q-table ahora es más grande
Qtable = np.zeros((500, 6)) # 3,000 celdas (vs 64)

# 3. Ajustar parámetros (más episodios, más exploración)
n_episodios = 25_000 # más episodios: espacio más grande
tasa_aprendizaje = 0.7
gamma = 0.95
decay_rate = 0.0005

# 4. ¡El resto del código es exactamente igual! ✅
Qtable_taxi = entrenar(n_episodios, min_epsilon, max_epsilon,
                   0.0005, env, 200, Qtable, 0.7, 0.95)

🎯 Este es el punto pedagógico clave del capítulo: Q-Learning es un algoritmo general. El mismo código que resolvió FrozenLake (16 estados) resuelve Taxi (500 estados) sin cambiar la lógica. Solo escala la tabla.

🔮 La limitación que viene

FrozenLake tiene 16 estados, Taxi tiene 500. ¿Y si tuviéramos 10,000,000,000 de estados, como en un juego de Atari? Una Q-table tendría que tener miles de millones de filas — imposible. Por eso en el Módulo 3 introducimos las redes neuronales: Deep Q-Learning.

2.5 & 2.6 · Errores frecuentes

Errores comunes y cómo resolverlos 🔧

Si tu agente no aprende bien, revisa estos problemas típicos:

❌
El agente siempre obtiene 0 de recompensa
Causas: ε demasiado alto (siempre explora al azar), alpha demasiado bajo (aprende muy lento), pocos episodios. Solución: revisa que decay_rate no sea 0 y aumenta n_episodios.
❌
Error "gymnasium not found"
El curso usa gymnasium (con "y"), no gym (la versión antigua). Instala con: pip install gymnasium.
❌
La Q-table no cambia después del entrenamiento
Asegúrate de que estás pasando la Qtable a la función entrenar() y guardando el resultado: Qtable = entrenar(...).
❌
El agente de Taxi siempre choca con paredes
Normal al principio — con ε = 1.0 explora aleatoriamente. Después de ~5,000 episodios debe aprender a evitarlas con la penalización de −1 por paso.
