🔮 La pregunta del módulo

3.1 · El Problema

Atari: el estado es una imagen 🎮

Los entornos de Atari no describen el estado con 8 números. El estado es directamente la imagen en pantalla — una cuadrícula de píxeles con colores:

Space Invaders 👾 (vista simplificada)

Dimensiones reales de Atari

Pantalla original: 210 × 160 píxeles
Canales de color: 3 (Rojo, Verde, Azul)
Valores por píxel: 0 a 255

El tamaño del espacio de estados

256(210 × 160 × 3) = 256100,800
≈ 10241,920 estados posibles
Para comparar: hay ~1080 átomos en el universo observable.

💡 La solución

En lugar de guardar el valor Q de cada estado en una tabla, usamos una red neuronal que estima el valor Q de cualquier estado sin haberlo visto antes. Es la diferencia entre memorizar y entender.

3.1 → 3.2 · La solución

La idea: reemplazar la tabla por una red neuronal 🧠

El salto conceptual es elegante: en lugar de una tabla de consulta, entrenamos una red neuronal que aprende la función Q.

Q-Learning clásico
(s, a) → 📋 tabla → número
Busca en la tabla el valor exacto del par (estado, acción). Si nunca vio ese par, no sabe.
Funciona hasta ~miles de estados
Deep Q-Learning (DQN)
s → 🧠 red → [Q(s,a₁), Q(s,a₂)…]
Pasa el estado por la red y obtiene los valores Q de todas las acciones a la vez. Generaliza a estados nuevos.
Escala a billones de estados

🔑 Diferencia clave: La red neuronal no "recuerda" cada estado — aprende patrones. Igual que un jugador experto de ajedrez que no memorizó cada posición posible, sino que aprendió a reconocer buenas y malas posiciones.

3.2 · DQN vs Q-Learning

¿Cómo aprende DQN? La función de pérdida 📉

En Q-Learning clásico actualizamos la Q-table directamente. En DQN usamos descenso de gradiente para ajustar los pesos de la red:

Q-Learning clásico

Actualización directa de la tabla:

\[ Q(s,a) \leftarrow Q(s,a) + \alpha[\text{objetivo} - Q(s,a)] \]

Sin función de pérdida. Sin gradientes.

Deep Q-Learning (DQN)

Se define una pérdida y se minimizan los pesos:

\[ \mathcal{L} = \big(Q_{\text{objetivo}} - Q_\theta(s,a)\big)^2 \]

Descenso de gradiente ajusta θ para minimizar L.

El ciclo de entrenamiento de DQN tiene dos fases:

1
Muestreo: el agente interactúa con el entorno y guarda experiencias (s, a, r, s') en una memoria.
2
Entrenamiento: se toma un lote aleatorio de la memoria y se hace un paso de descenso de gradiente.
3.2 · Inestabilidad

El problema: DQN es inestable sin cuidado ⚠️

Combinar una red neuronal con bootstrapping puede generar oscilaciones o divergencia durante el entrenamiento. Por eso DQN introduce tres soluciones:

💾

① Memoria de Repetición

Guarda experiencias pasadas y las reutiliza. Evita que la red olvide.

🎯

② Red Objetivo Fija

Una copia congelada de la red para calcular el objetivo TD. Evita que el objetivo se mueva constantemente.

⚖️

③ Double DQN

Usa dos redes para separar "qué acción elegir" de "cuánto vale". Evita sobreestimar Q-values.

3.2 · Memoria de Repetición

① Memoria de Repetición: no desperdiciar experiencias 💾

En RL clásico, cada experiencia se usa una vez y se descarta. DQN las guarda en un búfer de repetición y las reutiliza muchas veces:

BÚFER DE REPETICIÓN (capacidad N)
Experiencias almacenadas:
Cada barra = un tuple (s, a, r, s')
↑ muestrea lote aleatorio para entrenar

Ventaja 1: reutilizar experiencias

Una experiencia del nivel 1 puede usarse para entrenar mientras el agente está en el nivel 10. Aprende mucho más con el mismo número de interacciones.

Ventaja 2: eliminar correlaciones

Si entrenaras con experiencias consecutivas (paso 1, 2, 3…), la red aprendería a "esperar" la siguiente experiencia. Muestreando al azar del búfer, rompemos esa correlación.

🧠 Analogía: Es como estudiar con fichas de repaso mezcladas al azar, en lugar de leer el libro de principio a fin cada vez. Aprendes más rápido y retienes mejor.

3.2 · Red Objetivo Fija

② Red Objetivo Fija: atrapar una vaca quieta 🐄

El problema sin red objetivo: usamos la misma red para calcular el valor Q actual y el objetivo TD. Eso significa que el objetivo se mueve en cada paso de entrenamiento.

Analogía original del curso de HuggingFace 🤗

🤠
La estimación Q
(tratando de acercarse al objetivo)
persigue →
🐄
El objetivo TD
(que también se mueve porque usa la misma red)

Resultado: el entrenamiento oscila sin converger — como tratar de atrapar una vaca que también se escapa cada vez que das un paso.

🧠
Red Principal
Se actualiza en cada paso. Elige acciones y se entrena.
cada C pasos → copia de pesos
🧊
Red Objetivo
Congelada entre actualizaciones. Solo calcula el objetivo TD.
3.2 · Double DQN

③ Double DQN: evitar la sobreestimación ⚖️

DQN estándar puede sobreestimar los valores Q al principio del entrenamiento, lo que desestabiliza el aprendizaje. Double DQN lo resuelve separando dos tareas:

El problema con DQN estándar

Para calcular el objetivo TD: y = R + γ · maxa' Qobjetivo(s', a')

Usamos la misma red para elegir qué acción tomar en s' y calcular cuánto vale esa acción. Si la red tiene un error, amplifica ese error al maximizar el Q-value.

La solución de Double DQN: separar las dos responsabilidades entre las dos redes que ya tenemos:
🧠
Red Principal → elige qué acción tomaríamos en el estado s'
→ acción* 🧊
Red Objetivo → calcula cuánto vale tomar esa acción específica
→ Q-valor
🎯 En una frase

Double DQN dice: "Una red decide qué hacer, la otra evalúa qué tan bueno es." Al separar estos dos roles, se reduce drásticamente la sobreestimación y el entrenamiento se estabiliza.

Resumen

DQN: el algoritmo completo de un vistazo 🗺️

1
Preprocesar los frames (84×84, escala de grises) y apilar 4 consecutivos → estado s
2
Elegir acción con ε-greedy: pasar s por la Red Principal → tomar acción a
3
Ejecutar la acción → recibir recompensa r y nuevo estado s' → guardar (s,a,r,s') en el búfer
4
Muestrear un lote aleatorio del búfer → calcular el objetivo TD con la Red Objetivo
5
Calcular la pérdida L = (objetivo − Q estimado)² → paso de descenso de gradiente sobre la Red Principal
6
Cada C pasos: copiar pesos de la Red Principal a la Red Objetivo → volver al paso 2

Glosario

Glosario de los capítulos 3.1 y 3.2

Métodos tabulares Algoritmos como Q-Learning que representan la función Q en una tabla. Solo funcionan cuando el espacio de estados es pequeño y discreto.
DQN Deep Q-Network. Usa una red neuronal profunda para aproximar la función Q en lugar de una tabla. Funciona con entornos de imagen como Atari.
Función de pérdida (Loss) Número que mide cuánto se equivoca la red: L = (Q objetivo − Q estimado)². El entrenamiento consiste en minimizarla.
Descenso de gradiente Técnica para ajustar los pesos de la red en la dirección que reduce la pérdida. El "motor" del entrenamiento de cualquier red neuronal.
Búfer de repetición Memoria que guarda experiencias (s, a, r, s') pasadas para reutilizarlas en el entrenamiento. Evita olvidar y reduce correlaciones.
Red objetivo (Target Network) Copia congelada de la red principal que solo se actualiza cada C pasos. Sirve para calcular el objetivo TD de forma estable.
Red principal (Online Network) La red que se entrena en cada paso. Elige las acciones y es la que termina siendo el agente final entrenado.
Double DQN Variante de DQN que usa la red principal para elegir la acción y la red objetivo para evaluar su valor. Reduce la sobreestimación de Q-values.
Sobreestimación de Q Error sistemático donde la red asigna valores Q demasiado altos a acciones subóptimas al inicio del entrenamiento. Double DQN lo corrige.
