class Policy(nn.Module):
  def __init__(self, s_size, a_size, h_size):
    super(Policy, self).__init__()
    self.fc1 = nn.Linear(s_size, h_size)
    self.fc2 = nn.Linear(h_size, a_size)

  def forward(self, x):
    x = F.relu(self.fc1(x))
    x = self.fc2(x)
    return F.softmax(x, dim=1) # probabilidades que suman 1

🔑 Categorical: PyTorch tiene una distribución especial para acciones discretas. Con m = Categorical(probs) podemos muestrear una acción con m.sample() y obtener su log-probabilidad con m.log_prob(accion) — exactamente lo que necesita REINFORCE.

4.3 · Bug Educativo

El bug del método act: ¿puedes encontrarlo? 🐛

El notebook original de HuggingFace introduce intencionalmente un bug en el método act() para que practiques la depuración. ¿Lo encuentras?

def act(self, state):
  state = torch.from_numpy(state).float().unsqueeze(0).to(device)
  probs = self.forward(state).cpu()
  m = Categorical(probs)
  action = np.argmax(m) # ← ¿ves el problema?
  return action.item(), m.log_prob(action)
🔍 Pista

Recuerda el Cap. 4.1: una política aleatoria debe muestrear una acción de la distribución, no elegir siempre la más probable. np.argmax(m) no hace eso — y además, devuelve algo que no es un Tensor, lo que rompe m.log_prob(action).

✅ La corrección:
  action = m.sample() # muestrea de la distribución → Tensor ✓
  return action.item(), m.log_prob(action) # ahora funciona

Con m.sample() el agente no siempre elige la acción más probable — explora, y además devuelve un Tensor compatible con log_prob.

4.3 · Retorno hacia atrás

El truco del retorno calculado hacia atrás ⏪

En REINFORCE necesitamos calcular el retorno Gt para cada paso del episodio — pero solo las recompensas futuras cuentan, no las pasadas. El notebook usa un truco eficiente:

Episodio de 4 pasos: recompensas [+1, +1, +1, +1], γ=1.0

G=4
t=1
R=+1
G=3
t=2
R=+1
G=2
t=3
R=+1
G=1

Las acciones tempranas (t=0) tienen más peso (G=4) porque afectan más recompensas futuras. Las tardías (t=3) solo tienen su recompensa propia (G=1).

# Calcular retornos hacia atrás (eficiente en O(N))
returns = deque(maxlen=max_t)
for t in range(n_steps)[::-1]: # ← de atrás hacia adelante
  disc_return_t = (returns[0] if len(returns) > 0 else 0)
  returns.appendleft(gamma * disc_return_t + rewards[t]) # G_t = r_t + γ·G_{t+1}
4.3 · Función de Pérdida

Minimizar la pérdida = maximizar el retorno 📉→📈

PyTorch usa minimización de pérdidas. Para convertir nuestro ascenso de gradiente en descenso, simplemente negamos la función objetivo:

# Estandarizar los retornos (más estabilidad en el entrenamiento)
returns = torch.tensor(returns)
returns = (returns - returns.mean()) / (returns.std() + eps) # normalizar

# Calcular la pérdida de la política
policy_loss = []
for log_prob, disc_return in zip(saved_log_probs, returns):
  policy_loss.append(-log_prob * disc_return) # ← negativo: min pérdida = max retorno
policy_loss = torch.cat(policy_loss).sum()

# Paso de actualización
optimizer.zero_grad()
policy_loss.backward() # calcula los gradientes
optimizer.step() # actualiza los pesos θ
🔢 ¿Por qué estandarizar?

Si un episodio da G=500 y otro G=2, los gradientes serán muy diferentes en magnitud. Restar la media y dividir entre la desviación estándar hace que todos los episodios contribuyan por igual al aprendizaje.

➖ El signo negativo: Queremos maximizar J(θ) = E[G]. Como PyTorch minimiza, escribimos la pérdida como −log π(a|s) × G. Minimizar esto es equivalente a maximizar log π(a|s) × G.

4.3 · PixelCopter

PixelCopter: el mismo código, más dificultad 🚁

El mismo algoritmo reinforce() entrena al helicóptero. Solo cambian los hiperparámetros (red más grande, más episodios) porque el entorno es más complejo:

# PixelCopter tiene 7 variables de estado vs 4 de CartPole
# Necesita una red neuronal más grande
pixelcopter_params = {
  "h_size": 64, # 4× más grande que CartPole
  "n_training_episodes": 50_000, # 50× más episodios
  "max_t": 10_000,
  "gamma": 0.99, # sí usamos descuento aquí
  "lr": 1e-4, # tasa mucho más pequeña (más estable)
}

# Para PixelCopter usamos una red de 3 capas (más profunda)
class Policy(nn.Module): # nueva versión para PixelCopter
  def __init__(self, s_size, a_size, h_size):
    self.fc1 = nn.Linear(s_size, h_size)
    self.fc2 = nn.Linear(h_size, h_size * 2) # capa extra
    self.fc3 = nn.Linear(h_size * 2, a_size)

⏱️ 50,000 episodios tarda ~60 min en Colab GPU. Puedes bajar n_training_episodes a 10,000–20,000 para una versión más rápida — el agente no llegará a la meta de certificación (≥5) pero sí aprenderá a volar correctamente. Para la certificación necesitas los 50,000 completos.
